HTML

  • HTML stands for Hypertext Markup Language, which give the content structure.
  • HTML is one of the three cornerstone languages used in web development.
  • HTML is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • An Element is a type of content on a webpage
  • Text elements, Image elements, Header elements, and List elements are all element types commonly used to add content to pages.
  • Meta-related elements, Content division elements, and Semantic elements are some common element types that are used to divide and describe a page.
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage to users. Although, metadata is not seen by users; it contains important information about accessibilty, search engines and performance.
  • The head element is a container for metadata.
  • The head element conatins information about the webpage.
  • The charset element sets our character encoding
  • The http-equiv element sets an HTTP header for content
  • The viewport element sets the default view for our webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for a webpage.
  • The style element defines the CSS styles associated with a webiste
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewpoint.
  • The script element embeds client-side scripts in the HTML document
  • Elements that are nested inside other elements are called child elements
  • The element that holds all the children elemts is known as the parent element, which is the body element.
  • This "Parent/Child" language is often used by developers to explain the relationship among elements in HTML
  • When working with visible HTML elements, the content is contained between an opening tag and closing tag.
  • Remember: An element refers to the entire element from opening tag to closing tag. A tag refers only to what appears inside the angle brackets.
  • An HTML attributes defines additional information about an element
  • HTML attributes are always defined in the beginning element, never the closing element
  • The 'h1', 'h2', 'h3', 'h4', 'h5', and 'h6' elements represent the level of heading a given text block represents
  • Headings are larger or more prominent titles and subtitles to organize a page.
  • Order matters! When using headings, it is important that we use heading tags in order from 'h1' to 'h6' to make the page accessible to users whon access the page using screen readers or other devices.
  • The biggest heading, 'h1', should only appear once each page.
  • The 'P' element represents a paragraph or block of text
  • The 'ul', 'ol', and 'li' elements represents unordered lists, ordered lists, and list items. Basically, these HTML elements represents bulleted or numbered lists.
  • The 'img' element contains information about images that are displayed on the webpage. This element can contain various attributes. The 'src' attribute is the most important because it defines the location of the image.
  • The 'alt' attribute contains a text string that describes the appearance and functionality of the image.
  • The 'br' element creates a line of empty space, or a line break, between two blocks of content
  • The 'a' element (which stands for anchor) creates links to the same webpage or other webpages. The 'href' attribute points to the URL for the link
  • The footer element appears at the bottom of the page and ususally contains the author, copyright, contact, sitemap, and navigation
  • The section elements are semantic elements that define a section in a document that contains thematic content.
  • HTML uses the 'id' attribute as a way to identify elements and select them.

CSS

  • Cascading Style Sheets (CSS) is a programming language used by developers to define how webpage content is presented to users
  • CSS defines the color, aesthetics, visual layout, and organization
  • Incline CSS, internal CSS style sheet, and external CSS style sheet is three ways to style a webpage using CSS.
  • When a developer wants to style an element directly in the HTML file, they use incline CSS.
  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only
  • Incline Style sheets are useful for quick and permanent changes but are less flexible because each incline style must be edited individually whenever you make a design change.
  • It is possible to embed an entire stylesheet directly in the HTMLfile using 'style' element in the 'head' element.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file
  • The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code
  • External CSS style sheet is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier
  • When using an external style sheet, we link the file using a 'link' element placed in the HTML's 'head'
  • A selector defines the element or attributes to which the rules, or declaration, will apply
  • The CSS property we want to apply and the value of the property are two important components of declarations
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Every HTML element is essentially a box. Each box consists of margins outside the element,and the element's borders, padding, and content. Sometimes referred to as the CSS box model
  • Sometimes, we want to apply a rule to all the elements visible on the page. To do that, we use a special wild card symbol or '*'
  • You can combine two selectors into a single CSS rule with a comma, allowing the declarations to afeect both selectors.
  • Two or more elements that will share the same styling can be grouped together
  • A 'link' element creates the connection to the external file
  • The 'rel' attribute specifies the relationship between the current document and the linked document resource
  • The 'href' attribute specifies the location (URL) of the external source. The 'href' attribute is using a relative path
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector
  • Then the class attributes can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule
  • The box shadow CSS property adds shawdow effects around an element's frame. You can set multiple effects separated by commas
  • A box shadow is decribed by X and Y offsets relative to the elements, blur and spread radius,and color
  • Asssigning colors in CSS can be done in many ways, including by semantic references such as 'gray' or 'blue'
  • We can also refer to a color by its hexadecimal code, or hex#

Git

  • Git is a software on your computer that tracks changes that you make to your code
  • Github is a cloud based service that allows developers to store, manage, and collaborate on code.
  • Github is connected to Git
  • Repository ("Repo"): 1. Every project on GitHub is called a repository. 2. A folder for your project that contains all of the files and their revision history
  • The branch is a workspace where you can make changes that dont affect the live site
  • Git commands always starts with 'git' and are used to save and track versions of your code.
  • The checkout command: is to move the working branch to a new branch.
  • The '-b' flag: creates a new branch
  • It is good practice to name the branches for the feature that will be developed by them to help indicate the purpose of each branch.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • The add command: adds modifications in the current working branch, to the staging area.
  • The '-a' flag: indicates that we want to add all changes
  • The staging area informs Git that the damages will be added in the next commit
  • The pull command: to recieve a branch's modifications into the local environment
  • The 'orgin' keyword indicates the source of the pull will be in the GitHub Repo
  • The 'main' keyword indicates the branch
  • It is important to always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work. Otherwise, you might be working on outdated code!!!
  • The "push' command allows you to create a new remote branch or add new changes to an existing branch
  • git init: intialize an existing directory as a Git repository
  • git clone [url]: retrieve an entire respository from a hosted location via URL
  • git add [file]: add a file as it looks now to your next commit (stage)
  • git reset [file]: unstage a file while retaining the changes in working directory
  • git diff: diff of what is changed but not staged
  • git diff --staged: diff of what is staged but not yet committed
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot
  • git branch: list your branches a * will appear next to the currently active branch
  • git merge [branch]: merge the specified branch's history into the current one
  • git log: show all commits in the current branch's history
  • git branch [branch-name]: create a new branch at the current commit
  • git log: show the commit history for the currently active branch
  • git stash: save modified and staged changes
  • git pull: fetch and merge any commits from the tracking remote branch

JavaScript

  • JavaScript (JS) is a high-level dynamic programming language that allows users to interact with websites
  • USing JavaScript you can power your websites with effects, interactivity, data communication, and much more
  • JavaScript was invented by Brendan Eich
  • JavaScript is case-sensitive
  • You can declare a variable and give it a value on the same line
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • In JavaScript, to declare a variabe, we use "var" keyword
  • After "var" keyword, we give the variable a name
  • An assignment operator "=" to assign the specific data that we want stored in our variable
  • All strings are surrounded by quotation marks
  • Each line on JavaScript code ends with a semicolon
  • To access a string, you can simply refer to the varaiable's name
  • "console.log" method: to output a message to the web console by adding an agrument
  • The word "console" refers to a test environment that developers use to check out their code
  • The second part, ".log()" is a method, which measn a set of insructions that can be executed by a computer
  • The ".log()" method will output whatever we add inside the parentheses to the console
  • In JavaScript, A value is considered falsy if it is one of the following values: 0, -0, on, " ", null, undefined, Nan, or a Boolean data type with the value of false
  • All other values, including a Boolean with the value of true, are considerded truthy
  • The strict equality operator (===) checks to see if two values are equal, and returns a Boolean result "true" if the values are equal and false if the values are not equal
  • An array is a single variable that is used to hold a group of data
  • To create an array, we declare a variable in the same way that we declared a variable with onle one data item. Instead we assign the variable to a group of values.
  • Using each piece of data's index, or location number, we can access individual data inside the array
  • The first item in any array has an index of 0, not 1
  • The indices, or location identifie, will always be whole numbers that increases by 1 each time
  • A "for" loop uses the predictable pattern of indicies to perform a task on all the items in array by allowing a single code block to be executed over and over
  • The array length is important because the "for" loop needs to know how many times the code will run until there are no items left in the array
  • A "for" loop contains three important statements
  • The first statement determines the starting point for our loop. Since we want to start at the beginning of the array, we declare a variable x and give it a value of 0 (because the first index of an array is always 0)
  • The second statement is the condition. As long as the number in the variable x is less than the length of the array, the loop will keep running. This will allow the loop to execute one time for each item in the array. We use the length built-in array property on our shapes array to express that condition
  • The final statement is what allows the array to iterate over each item. Since indices follow a predictable pattern, we know the number increases by 1 each time. The code x++ is a shortcut way of writing x+1. Now every time the code loops, x will increase by 1
  • A "for" loop starts with "for" when writing a code
  • We put the code that we want to be executed for each loop inside curly brackets
  • Commenting out code makes it inactive without deleting it, just in case you need it later. To comment out multiple lines of JavaScript code, put /* before and */ after the block of code
  • A function is simply a set of instructions that tells the computer how to perform a certain task
  • A function is different from a "for" loop or conditional statement because functions do not automatically execute when the JavaScript file is run
  • You must call a function by name in order to execute it
  • Being able to call a function to execute it also means you can control where in the control flow a task is executed, or if it is executed at all
  • To create a function, we use the "function" keyword
  • A function's name is followed by parentheses ( )
  • The code to be executed is written inside the curly brackets { }
  • Remember to include the empty parentheses after the function name. Note that it is common practice to indent the "for" loop code below the function name, within the curly brackets, to add visual structure to your code
  • When we declare a function in our JavaScript code, we are creating a reusable block of code that performs a specific task
  • However, declaring a function does not mean that the code will automatically execute. We need to call the function in the code at the spot we want it to execute
  • When we create a function, we are defining a block of code that performs a specific task and giving it a unique name
  • It is also important to remember that since control flow dictates that JavaScript files are executed from top to bottom, order matters.
